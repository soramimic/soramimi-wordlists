# ADR 00002: pokemon.csv のフォーム収録・表記ゆれ・タイプ列

- Status: accepted
- Date: 2026-07-05
- Supersedes: none
- Superseded by: none

## Context

pokemon.csv は種(全国図鑑順)のみを収録していたが、「ライチュウ（アローラのすがた）」のようなリージョンフォームも空耳の語彙として使いたい。フォームは呼び方が複数あり(ライチュウ / アローラライチュウ / ライチュウ（アローラのすがた）)、どれを表層・発音として採用するかは利用側で選べるようにしたい。また空耳変換の絞り込み用にポケモンのタイプ(でんき等)も持たせたい。

## Decision

- 列は `id, original, surface, pronunciation, type1, type2` とする。type1/type2 はポケモンのタイプの日本語名(PokéAPI ja-Hrkt)。単タイプは `type2=NA`
- 収録するフォームは、名前として独立して呼ばれるものに限定する
  - 地方のすがた: フォーム名が「アローラ/ガラル/ヒスイ/パルデアのすがた」に完全一致するもの
  - キョダイマックスのすがた
  - メガシンカ(`is_mega`)。フォーム名自体が完結した名前(メガリザードンX 等)なので 1 行のみ
  - フォルム違い・ミノ違い・オスメスのすがた・いちげきのかた等は収録しない(単体で名前として呼ばれない)
- 「〇〇のすがた」系は表記ゆれ 3 行を同一 id で持つ。「すがた抜き」でも「すがた込み」でも使えるようにするため:

  ```csv
  id,original,surface,pronunciation,type1,type2
  25,ライチュウ,ライチュウ,ライチュウ,でんき,NA
  1038,ライチュウ（アローラのすがた）,ライチュウ,ライチュウ,でんき,エスパー
  1038,ライチュウ（アローラのすがた）,アローラライチュウ,アローラライチュウ,でんき,エスパー
  1038,ライチュウ（アローラのすがた）,ライチュウ（アローラのすがた）,ライチュウアローラのすがた,でんき,エスパー
  ```

- **id はポケモン単位で振る。** 種と各フォームは別ポケモン(図鑑上も別)なので別 id、同一フォームの表記ゆれ 3 行は同じポケモンなので同一 id。soramimic の実装(js/soramimic.js、editor の lib/soramimic.js)は使用済み単語の判定を id で行っており、これを維持する前提。ライチュウとアローラライチュウは同じ歌詞に共存でき、同一フォームの表記ゆれ同士は排他になる
- 括弧は公式表記に合わせ全角「（）」。発音は括弧を除いて連結し、かなはソースのまま(「のすがた」はひらがな維持)
- 表記の正規化: 全角英数記号は NFKC で半角へ(ポリゴン2、タイプ:ヌル)。♀♂は surface では表記を残し、pronunciation のみメス/オスに置換(ニドラン♀ → 発音ニドランメス)
- id は種が全国図鑑 No−1(0〜)、フォームは種の後ろから連番。フォームの id は世代追加時に振り直されるため、**永続キーには使わない**
- 同名フォームの重複(パルデアケンタロスの 3 品種等)は 1 つに統合し、タイプは最初のフォームのものを使う

## Consequences

- 種 1025 + フォーム 178(2026-07 時点)が収録され、フォームはタイプ絞り込みでも本体と別のタイプ(アローラライチュウ=でんき/エスパー)として扱える
- 「ライチュウ」という発音の行が種とフォームの両方に存在する(別 id なので両方採用されうる)。気になる場合は利用側で pronunciation の重複を除くこと
- キョダイマックスにも一律で「キョダイマックス+種名」の口語行を生成している。不自然なら SUGATA_FORM_NAMES から除外するだけで消せる
- 新世代・新フォームは `tools/update_pokemon.py` の再実行(年次バッチ、手動実行も可)で自動反映される
