# ADR 00003: nations.csv の表記固定・ISO マップ・status 列

- Status: accepted
- Date: 2026-07-05
- Supersedes: none
- Superseded by: none

## Context

nations.csv の 193 行は国連加盟国と 1 対 1 対応しているが、表記は「アメリカ」「イギリス」「グルジア」のような独自の通称で、ソース(mledoze/countries の日本語名)とは一致しない。名前だけでは機械的に突き合わせられず(「コンゴ」が 2 行、「ギニア」に赤道ギニアが含まれる等)、自動更新でソースの表記に置き換えると既存の使い勝手と id が壊れる。また、国の改名(グルジア→ジョージア)や国連脱退・消滅が起きたときに、古い名前を消すのではなく「現存するか」を区別して残したい。複数の読み(アメリカ/アメリカ合衆国/米国)も将来持てるようにしたい。

## Decision

- **既存行の original / surface / id は自動更新で一切書き換えない。** ソースとの対応は `tools/nations_map.csv`(ISO 3166-1 alpha-3 cca3 → id)で管理する。マップは初回に全 193 国分を人手確認して固定した(コンゴ 2 行は COG=110/COD=118、id 179 のギニアは GNQ=赤道ギニア)
- `status` 列を追加する。値は `current`(現存・国連加盟)/ `former`(旧称・脱退・消滅)
- 自動更新(`tools/update_nations.py`)がやるのは次の 3 つだけ
  - 新規加盟国を末尾に追記(id は連番、表記はソースの日本語通称、status=current)
  - 加盟国でなくなった国の行を status=former に変更(行は消さない)
  - 再加盟したら status=current に戻す
- **別名・旧称・複数読みは同じ id の行を手動で追加する**(例: id=55 に「ジョージア」current と「グルジア」former、id=190 に「アメリカ」と「アメリカ合衆国」を両方 current で)。自動更新が status を管理するのはマップの original と一致する行だけなので、同 id の手動行は保護される。国の改名時はマップの original を新表記に更新すること(そうしないと旧表記の行が加盟国として current に戻される)
- Wikidata の ja エイリアスによる別名の自動取得は見送る。「米国」「USA」のような読み上げに向かない表記が多く、フィルタの整備コストがノイズ除去の価値を上回らないため。必要になれば候補生成+人手確認の半自動として別 ADR で設計する
- ソース異常への防御として、加盟国数が 180〜210 を外れた場合と増減が 5 か国を超えた場合はエラーで中断する

## Consequences

- 国連加盟国の増減は年次バッチで自動反映され、既存の表記・id・手動で足した別名行は壊れない
- 利用側は `status=current` で絞れば現存国のみになる(status を見ない場合は旧称も候補に含まれる)
- 新規加盟国の表記はソースの日本語通称がそのまま入るため、既存の通称スタイルと合わない場合は PR レビューで手直しする(マップの original も合わせて直すこと)
- 国の改名はソースからは機械判定できないため、従来どおり人が気づいて手動対応する(新表記の行を追加し、旧表記を former にする)
