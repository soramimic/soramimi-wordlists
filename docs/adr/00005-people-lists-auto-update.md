# ADR 00005: 人名リスト(baseball/football/physicist)の自動更新

- Status: accepted
- Date: 2026-07-05
- Supersedes: 00001 のうち baseball/football/physicist を自動更新対象外とした判断
- Superseded by: none

## Context

ADR 00001 では野球選手(やきゅうた広場提供)・サッカー選手(ヨロスー様提供)・物理学者(手選び)を自動更新の対象外としていた。stations での「Wikipedia 冒頭文の定型パターンから読みを取る」方式が実証できた(ADR 00004)ため、人名リストにも同じ要領を適用する。人名では記事冒頭が「姓 名(せい めい、」形式のため、**姓名の分割と分割済みの読みが同時に取れる**(従来の NameDivider 推定+目視確認より確実)。一方で、既存リストは歴代選手の名鑑・手選びのキュレーションとして価値があるため、置き換えではなく保持する。

## Decision

共通方針(`tools/wpnames.py` に共通処理を実装):

- **既存行は一切書き換えず、未収録の人だけを追記する。** 既存データ(やきゅうた広場の歴代選手表・ヨロスー様のデータ・手選びの物理学者)はそのまま正であり、クレジットも維持する
- 姓名と読みは記事冒頭「姓 名(せい めい、」から取得。「本名:姓 名〈読み〉」(登録名が記事名のケース)、台湾人選手等の「(カタカナ・カタカナ」、カタカナ名(名前=読み、姓は最後の区切り)に対応
- 既存との照合は氏名の表層(空白除去+異体字正規化: 髙/高、﨑/崎 等)。**リンク先が同姓同名の別人記事**である事故を防ぐため、冒頭文に競技名(野球/サッカー)が無ければスキップして報告する
- 記事が未作成の新人はスキップし、次回実行時に記事ができていれば拾う。読みを機械決定できない名前(ラテン文字イニシャル入り等)はスキップして報告し、必要なら手動追加する

リスト別:

- **baseball**: 12球団の「Template:〇〇の選手・スタッフ」の支配下・育成セクションから現役選手を取得。登録名はテンプレートのリンク表示(`[[本名|登録名]]`)と本名記載から取り、type=registered 行を生成。新規行は既存の新しい規約(id=`氏名_球団_00000`、読みは「姓 名」空白区切りカタカナ)に合わせる
- **football**: 「Template:日本プロサッカーリーグ」を展開して J1〜J3 クラブを取得し、各「Template:〇〇のメンバー」の選手セクション(GK/DF/MF/FW)から取得。スタッフ・マスコット・関連情報は除外。既存規約(読みはカタカナ、外国人は family/given 行も surface=フルネーム)を踏襲
- **physicist**: Wikidata の職業=物理学者かつ **sitelinks>=20**(多言語版20版以上に記事がある著名層、約1,200人)を対象に拡大。読みは既存規約に合わせひらがな

付加情報(替え歌動画用):

- **画像**: stations と同様に `image`(Commons直リンク)/`image_page`(ライセンス確認先)列を持つ。Wikidata P18 から取得し、新規行に加えて既存行にも付与する。本人確定の方法: (a) 現役は ロースターのリンク先記事経由・Wikidata の物理学者集合との名前一致、(b) 引退選手等への遡及付与は `tools/enrich_images.py` で「氏名(+曖昧回避サフィックス)で記事を引き、曖昧さ回避ページを除外し、冒頭に分野キーワード(野球/サッカー/物理)があるもの」だけに限定する。画像は全て Wikimedia Commons 収蔵(再利用可能ライセンス)だが、CC BY-SA 等は利用時に image_page でクレジット条件を確認すること
- **baseball の team 列**: 新規行は追加時点の球団。既存選手が移籍していたら旧バージョンの形式に合わせ `-球団` をハイフンで追記する(例: `西武-ソフトバンク`)。既存行への変更はこの追記と空欄への画像付与のみ

## Consequences

- 新人・新外国人選手、著名物理学者が年次バッチで自動追記される(初回: 野球+261人、サッカー+943人、物理学者+871人)
- 移籍・引退は反映しない(歴代名鑑として追記のみ。team 列は追加時点の球団)
- 同姓同名の別人記事・記事未作成・読み不明はスキップされ、実行ログに「要確認」として出る。年次PRのレビューで判断する
- 野球の「選手表ニキ」フロー(README の手動手順)は残るが、日常の更新はバッチで足りる想定
