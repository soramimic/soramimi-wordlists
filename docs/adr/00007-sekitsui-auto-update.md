# ADR 00007: sekitsui(脊椎動物)の自動更新

- Status: accepted
- Date: 2026-07-07
- Supersedes: 00001 のうち sekitsui を「キュレーションのため自動更新対象外」とした判断
- Superseded by: none

## Context

ADR 00001 では sekitsui(脊椎動物)を physicist と並べて「有名どころを人が選ぶキュレーション」とみなし、自動更新の対象外とした。しかし実態は約7,280件の難読和名(ケナガルーセットオオコウモリ等)からなる**網羅志向のリスト**であり、「有名どころの手選び」という前提と食い違っていた。網羅リストであれば、公開データからの機械的な追記はリストの性格を壊さず、むしろ網羅性を高める。

sekitsui は `surface = pronunciation = カタカナ和名` という構造を持つため、人名リスト(ADR 00005)で最も厄介だった「記事冒頭文からの読み抽出」が不要になる。日本語ラベルがそのまま答えになる。

## Decision

Wikidata を出典に、`tools/update_sekitsui.py` の年次バッチで未収録の種を追記する。ADR 00001 の4基準を満たす:

1. **機械可読な構造化データ**: Wikidata の SPARQL(taxon)。HTML スクレイピングはしない
2. **認証不要で無人取得**: WDQS は認証不要
3. **ライセンス**: Wikidata のラベルは CC0。README のリスト一覧に出典を記載する
4. **差分方式**: 既存の網羅リストはそのまま正とし、未収録の和名だけを id 連番で追記する。既存行の表記・id は書き換えない

実装方針:

- 対象は **rank=種(Q7432)** で **日本語ラベルがカタカナのみ**の taxon。カタカナ以外(「総鰭類」等の漢字 label、曖昧さ回避付き)は既存リストの性格に合わないので除外する
- **化石種・絶滅種も含める**。Wikidata で rank=種 として登録されていれば対象にし、`extinct` 列(yes/no)で区別する。判定は IUCN ステータス(P141)が絶滅種(Q237350)/野生絶滅(Q239509)、または instance of(P31)が化石タクソン(Q23038290)のいずれか。これで近年の絶滅(ドードー等)と地質時代の化石種(ティラノサウルス等)の両方を拾う。どちらのシグナルも無い種は現生扱い(no)とする(ヒューリスティックなので取りこぼしはありうる)
- **`class` 列に大分類(魚類 / 両生類 / 爬虫類 / 鳥類 / 哺乳類)を持たせる**。列名は他リストの `type`(pokemon=タイプ、baseball=名前種別)と意味が被らないよう、自然な `class` にする。魚類は分割クエリの3綱(条鰭類・軟骨魚類・無顎類)をまとめて「魚類」とする。既存行にも Wikidata から分類を引いて付与し、引けなかった行は `NA` とする
- 脊椎動物 Q25241 を一括クエリすると WDQS がタイムアウトするため、綱ごとに分割してクエリする。硬骨魚類 Q27207 は系統樹(P171)上で四足動物まで含みうるため使わず、魚類は**条鰭類 Q127282・軟骨魚類 Q25371・無顎類 Q161095**に分ける。綱の系統樹は重複しうるが、`original` との照合で重複排除するので無害
- 対象の綱: 哺乳類 Q7377 / 鳥類 Q5113 / 爬虫類 Q10811 / 両生類 Q10908 / 条鰭類 Q127282 / 軟骨魚類 Q25371 / 無顎類 Q161095
- 取得総数が下限(5,000)を下回ったら取得失敗とみなして中断する(physicist の妥当性ガードと同様)

## Consequences

- sekitsui が pokemon / nations / stations / baseball / football / physicist に続く自動更新対象になる
- 化石種・絶滅種(アウストラロピテクス等の化石人類を含む)も rank=種 で登録されていれば追記され、`extinct=yes` になる(意図的に含める方針)。現生種のみで生成したい利用側は `extinct=no` で絞れる
- `class`/`extinct` 列の追加により既存の全行にも大分類・絶滅フラグが付く。Wikidata で分類を引けなかった既存行は `class=NA`(絶滅フラグはシグナルが無ければ `no`)になる(実行ログに件数を出す)。この列追加は既存の `original`/`surface`/`pronunciation` は書き換えず、情報を付加するのみ
- 綱 QID は Wikidata の分類体系に依存する。将来、綱の親子関係や QID が変わってタイムアウト・件数急減が起きた場合は、`CLASSES` の分割単位を見直す。件数の急減は妥当性ガード(下限5,000)で検出される
- 追記のみで、既存の和名の削除・改名の反映はしない(網羅リストへの追記として運用)
