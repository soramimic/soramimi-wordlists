# ADR 00009: physicist を広義の scientist リストに拡張

- Status: accepted
- Date: 2026-07-12
- Supersedes: 00005 のうち physicist(物理学者)の対象範囲・列設計の判断
- Superseded by: none

## Context

ADR 00005 で physicist.csv を「Wikidata の職業=物理学者 かつ sitelinks>=20」で自動追記する方針にした。しかし空耳作詞の題材としては物理学者に限る必然性がなく、化学・数学・天文・生物・計算機科学・地学まで含む広義の「科学者」の方が実用的で、網羅性も高い。また、替え歌動画や題材選びで「分野で絞る」「時代で絞る」「ノーベル賞受賞者だけ」といったフィルタ需要があるため、名前・読みに加えて分野・時代・生没・国・性別・ノーベル賞の付加列を持たせたい。

physicist.csv は物理学者に限定された名前空間で、拡張ではなくリスト自体の置き換え(リネーム)が自然である。既存の手選び+著名層(1,240人・2,494行)の読み・id・表記は正として保持する。

## Decision

physicist.csv を **scientist.csv にリネーム(置換)** し、`tools/update_scientist.py` で生成する。physicist.csv は廃止する。

**対象**: Wikidata の職業(P106)が次の7職業のいずれかで、sitelinks>=20(多言語版20版以上)の人物。

| P106 | field ラベル |
|---|---|
| Q169470 | 物理 |
| Q593644 | 化学 |
| Q170790 | 数学 |
| Q11063 | 天文学 |
| Q864503 | 生物学 |
| Q82594 | 計算機科学 |
| Q520549 | 地学 |

**列**: `id, original, surface, pronunciation, type, field, era, birth_year, nobel, gender, country, status, description, image, image_page`。旧 physicist.csv の `id/original/surface/pronunciation/type/image/image_page` はそのまま引き継ぐ。

- **field**(分野・単一列のスラッシュ区切り多値): 該当する職業の日本語ラベルを上表の優先順(物理→化学→数学→天文学→生物学→計算機科学→地学)で並べ、"/" で連結する(例 `物理/数学`)。切り詰めはしない(ロモノーソフのように5分野該当なら5つとも入る)。無ければ `NA`。CSV はカンマ禁止(利用側パーサが素朴な split(","))なので区切りは必ず "/"。**多値を1列で持てる理由**: ソラミミック本体に部分一致演算子 `~=` を追加した(別リポジトリ soramimic 側の対応)ので、`field~=物理` と書けば `物理/数学` のような多値セルにも一致し、分野で絞り込める。列を増やさずに複数分野を表現できる。app 側 setting.json での facet 定義・`~=` 実装は soramimic 側で対応する(本 PR の範囲外)。既存 physicist.csv 行で Wikidata と照合できなかった人は、出自が物理学者なので `field=物理`(Wikidata 一致があればそれを優先)。
- **description**(業績の短い完結文): 動画キャプション等で使う短いテキスト。Wikipedia 日本語版記事の冒頭(fetch_extracts)を元に、(1) 記事名直後の生没年・原語表記カッコ(例「（ドイツ語: … 1879 - 1955）」)を除去し、(2)「。」区切りで**完結した文だけ**を目安90字まで連結、(3) 常に「。」で終わるよう整える(1文が120字超のときのみ「、」境界で切る)。ハードカットで文の途中で切れるのを避け、「誰で・何で知られるか」が1〜2文で完結するようにする。無ければ Wikidata の ja description(schema:description、CC0)にフォールバック、どちらも無ければ `NA`。**サニタイズ**: ASCII カンマ・二重引用符は除去、改行・タブは空白に潰して1行にする(全角の「、」「。」「（）」「：」は許容)。利用側パーサ(素朴な split(","))を壊さないため必須。
- **era**(時代区分): 生年から古代(≤500)/中世(501–1500)/近世(1501–1700)/近代(1701–1900)/現代(1901–)。生年不明は `NA`。境界は生年 basis の明快な丸め値で、変更容易にするため `tools/update_scientist.py` の `era_of` に集約する。生年 basis のため、アインシュタイン(生1879)は業績が20世紀でも `近代` になる。これは既知の割り切り。
- **birth_year**: P569 の西暦年のみ。紀元前は `前287` 形式(Wikidata の格納値に従う)。不明は `NA`。
- **nobel**: 科学系ノーベル賞受賞者か `yes`/`no`。P166(受賞)に instance of(P31)が ノーベル賞(Q7191)である賞があれば yes(物理 Q38104 / 化学 Q44585 / 生理学・医学 Q80061 / 経済 Q47170 等)。既存で Wikidata と照合できなかった行は `NA`。
- **gender**: P21 → `男性`/`女性`、それ以外は `その他`、不明は `NA`。
- **country**: P27(市民権のある国)の日本語ラベル。情報列(facet フィルタ対象外)なので複数値は "/" 区切りのままでよい。出力順はソート固定(WDQS の GROUP_CONCAT 順が非決定的で年次 PR のノイズになるため)。ラベルにカンマが含まれる場合はカンマを除去(スペース化)してパーサを守る。不明は `NA`。
- **status**: P570(没年)があれば `物故`。無くても生年既知で(現在年−生年)>120 なら `物故`。生年既知で存命圏なら `存命`。生没年とも不明は `NA`。
- **image/image_page**: 既存値を保持し、空欄は Wikidata P18 から付与(update_physicist.py と同じ image_pair 方式、Commons 直リンクのみ、URL エンコードでカンマ対策)。

**取得方針(WDQS タイムアウト回避の2段構え)**:

1. **人物集合と分野**: 職業ごとに「P106∈集合 かつ sitelinks>=20」の QID・ja記事タイトルを取得し、Python で field トークンを集約する(7職業を一括集約すると WDQS がタイムアウトしうるため職業ごとに分割)。
2. **属性の一括取得**: 集めた QID を200件ずつのバッチにし、`VALUES ?p {…}` で P569/P570/P21/P27/P166/P18 と ja description(schema:description)を OPTIONAL + GROUP BY 集約で取得する。巨大な系統走査を避ける。description の主ソースは Wikipedia 冒頭文(fetch_extracts)で、これは全人物分を1度に取得して読み抽出(parse_person)と共用する。

読み・姓名分割は ADR 00005 と同じ(記事冒頭「姓 名(せい めい、」から抽出、日本人漢字名はひらがな、カタカナ名はそのまま、機械決定不能はスキップして「要確認」に報告)。

**妥当性ガード**: 取得できた科学者総数が 2,000 未満または 12,000 超なら失敗として中断する。

## Consequences

- physicist / baseball / football と同様、著名科学者が年次バッチ(update-wordlists)で自動追記される。分野・時代・生没・国・性別・ノーベル賞の付加列でフィルタ利用ができる。
- **破壊的変更**: physicist.csv を廃止し scientist.csv に置き換える。submodule で physicist.csv を参照している利用側は scientist.csv への追随が必要。ファイル名変更を周知する。
- **P106 依存の限界**: 職業タグに物理/化学等が付いた政治家・万能人(例: 学者かつ政治家)も混入しうる。sitelinks>=20 の著名層に限っており、題材リストとしては許容する。職業タグが無い科学者は取りこぼす(網羅性は Wikidata の P106 付与に依存)。
- **era は生年 basis の割り切り**: 業績の時代とはずれうる(アインシュタイン=近代)。境界値は `era_of` で一元管理し、見直しは容易。
- 列追加により既存の全行にも新列が付く。Wikidata と照合できなかった既存行は field=物理・他は NA になる(実行ログに件数を出す)。既存の original/surface/pronunciation/type/id は書き換えない。
- **field は単一列のスラッシュ区切り多値**: ソラミミックの部分一致演算子 `~=` に依存する。`~=` を実装していない利用側では `field` を完全一致で評価すると `物理/数学` が `物理` に一致しない点に注意(その場合は利用側でスラッシュ分割するか、`~=` 相当の部分一致で評価する)。分野数に上限はなく、実データの最大は5分野(ロモノーソフ・キルヒャー)。
