# ADR 00013: stations.csv の路線名(lines)列

- Status: accepted
- Date: 2026-07-08
- Supersedes: none
- Superseded by: none

## Context

ソラミミ動画の字幕で「伊達(だて) 福島県 JR東日本 東北本線」のように駅の路線を表示したい。stations.csv には所在地(prefecture/city)はあるが路線情報が無い。路線は「運営者略称 + 路線名」(JR東日本 東北本線、東京メトロ 丸ノ内線)の粒度が欲しい。

Wikidata の P81(接続路線)は中小の駅にはよく付いているが、東京・池袋などの主要駅ほど付いていない。一方 Wikipedia 日本語版の駅記事は駅情報テンプレート(所属事業者/所属路線)がほぼ全駅で整備されている。ただし記事の一括取得は本文が大きく、全駅に対して行うのは重い。

## Decision

- `lines` 列を追加する。値は「運営者略称 + 半角スペース + 路線名」を「／」で連結した文字列。運営者略称は Wikidata P1813(ja)、無ければ ja ラベル
- 取得は 2 段構え(`tools/enrich_lines.py`)
  - まず Wikidata: 駅の P81 → 路線の P137(運営者)。路線の運営者が複数のとき(東海道本線の JR3社+JR貨物等)は「駅自身の P137 に含まれる会社 > 貨物系以外 > 名前順」で 1 社を選ぶ。路線名が運営者名で始まる場合(長良川鉄道越美南線等)は運営者を付けない
  - Wikidata で取れなかった駅だけ Wikipedia 記事の駅情報テンプレートから補完(CC BY-SA 4.0。読みの抽出と同じ扱い)
- enrich_lines.py は既定で lines が空の行だけ埋める(手動修正を保護)。`--refresh` で全行再取得
- `tools/update_stations.py` は新規追記する駅の lines を同じロジックで埋める。既存行の lines には触らない
- どちらの情報源にも無い駅(樺太の旧駅、索道駅、計画駅など約180件)は空のままにする。利用側は lines が空でも動くこと

## Consequences

- 現役駅の約98%に路線名が入り、動画字幕にそのまま使える
- 値はスナップショットであり、路線の改称・移管(相鉄・東急新横浜線開業のような変化)は自動追従しない。年次更新時に `--refresh` を手動実行するか、気づいたら手で直す
- Wikipedia 由来の値が混ざるため、列全体のライセンスは CC BY-SA 4.0 側に合わせて扱うのが安全
